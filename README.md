# WASM Photo Lab

Веб-редактор изображений в духе Photoshop, собранный как браузерное SPA-приложение. Интерфейс написан на React + TypeScript, а пиксельные операции выполняются в Rust/WebAssembly внутри Web Worker.

## Что умеет приложение

- Вступительная страница на `/` с кратким описанием проекта.
- Рабочий редактор по адресу `/#editor`.
- Импорт одного или нескольких изображений.
- Photoshop-подобные слои: порядок, видимость, непрозрачность, заливка, режимы наложения, маски, обтравка, группы и блокировки.
- Инструменты: перемещение холста, трансформация фрагментов, кисть, карандаш, ластик, лассо/ножницы, прямоугольное выделение, волшебное выделение, кадрирование, пипетка, заливка и масштаб.
- Копирование или вырезание выделенной области в новый слой.
- Перемещение и масштабирование импортированных или вырезанных фрагментов.
- Экспорт в `PNG`, `JPEG` или `WebP`.
- Локальные снимки проекта в IndexedDB.
- Мобильный интерфейс, где боковые панели скрыты за перетаскиваемыми круглыми кнопками.

## Структура проекта

- `web` - фронтенд на React/Vite, canvas-интерфейс, инструменты, панель слоёв, локальное хранилище и Playwright-тесты.
- `wasm-engine` - Rust-ядро для документов, слоёв, масок, режимов наложения, фильтров, импорта/экспорта и истории.
- `shared-types` - TypeScript-контракты, общие для интерфейса и worker-а.
- `docker` - конфигурация nginx для раздачи production-версии SPA.
- `scripts` - вспомогательные скрипты для локального запуска и Docker.

## Требования

- Node.js 22+ рекомендуется.
- npm.
- Rust toolchain.
- `wasm-pack`.

Установить Rust/WASM-инструменты:

```bash
npm run check:toolchain
```

Установить npm-зависимости:

```bash
npm install
```

## Разработка

Сначала соберите WASM-пакет, затем запустите Vite:

```bash
npm run build:wasm
npm run dev
```

Открыть:

```text
http://127.0.0.1:5173/
```

Сначала открывается вступительная страница. Редактор доступен по адресу:

```text
http://127.0.0.1:5173/#editor
```

## Сборка

Собрать полную production-версию:

```bash
npm run build
```

Эта команда выполняет:

```bash
npm run build:wasm
npm run build:web
```

Готовые файлы лежат здесь:

```text
web/dist
```

## Временный статический запуск без dev-сервера

Для быстрой временной проверки можно собрать приложение с относительными путями к ассетам и открыть готовый HTML-файл:

```bash
npm run start:static
```

Этот скрипт запускает `scripts/open-static.ps1`, собирает WASM, собирает фронтенд с `--base ./` и открывает:

```text
web/dist/index.html
```

Важно: некоторые браузеры блокируют Web Worker или WASM при запуске через `file://`. Если это случилось, используйте `npm run dev` или Docker.

## Docker

Собрать образ:

```bash
npm run docker:build
```

Запустить контейнер:

```bash
npm run docker:run
```

Открыть:

```text
http://127.0.0.1:8080/
```

Ручные команды:

```bash
docker build -t wasm-photo-lab .
docker run --rm -p 8080:80 wasm-photo-lab
```

Docker использует отдельный этап сборки: устанавливает Rust, `wasm-pack`, npm-зависимости и собирает приложение. Этап запуска работает на nginx, поддерживает резервный переход на `index.html` для одностраничного приложения и отдаёт `.wasm` с правильным MIME-типом `application/wasm`.

На Windows npm-скрипты для Docker используют PowerShell-helper-ы из `scripts/`, чтобы находить Docker Desktop даже до обновления PATH в новом терминале.

## Карта кода

- `web/src/App.tsx`
  - состояние SPA-роута: вступительная страница или редактор;
  - состояние редактора: документ, инструменты, выделение, слои, мобильные панели;
  - рендер canvas и обработка pointer-событий;
  - импорт/экспорт, локальные снимки и команды слоёв/инструментов.
- `web/src/App.css`
  - стили вступительной страницы;
  - настольная раскладка редактора;
  - всплывающие настройки инструментов, наложения холста, панель слоёв;
  - мобильные плавающие кнопки и выдвижные панели.
- `web/src/engine/engineProxy.ts`
  - типизированный мост запрос/ответ от React к Web Worker.
- `web/src/engine/engine.worker.ts`
  - ленивая инициализация WASM и отправка команд в Rust.
- `wasm-engine/src/lib.rs`
  - модель документа, стек слоёв, маски, режимы наложения, фильтры, история и экспорт.
- `shared-types/src/index.ts`
  - общие объекты передачи данных и операции, которые TypeScript и Rust используют через JSON-вызовы.
